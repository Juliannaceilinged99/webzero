WebZero

A web server that serves 5,000 req/sec on a 2001 Pentium III.

C99 Linux + Windows XP < 80KB binary Zero malloc during serving Zero dependencies

Quick Start

# Build the server
make

# Build your site into a .web bundle
node tools/wz.js build examples/landing-page

# Serve it
./webzero examples/landing-page.web 8080

Philosophy

WebZero does one thing: serve static files insanely fast on minimal hardware.

The entire site is compiled into a single .web bundle — a binary file that gets mmap'd into virtual memory at startup. Every asset is pre-compressed with Brotli level 11. Every route is pre-indexed in a binary trie.

When a request arrives, the server does a trie lookup (1-3 pointer chases), then calls sendfile(). There is no parsing, no allocation, no string formatting.

Performance

5K req/sec on Pentium III 1GHz
70KB Linux static binary size
4MB total memory footprint
0 malloc calls during serving
